## 2. โครงเทิร์น — 2 เฟส (`js/turn.js`)

ทุกคนเล่นคนละ 1 เทิร์น/รอบ ตามลำดับ `s.order` · เทิร์นแบ่งเป็น:

### เฟส 1 — เฟสโจมตี (ไม่จับเวลา)
เลือกอย่างใดอย่างหนึ่ง:
- **ใช้ท่าโจมตี 1 ท่า** โดย "อ้างอาชีพ" (มีการ์ดจริงหรือโกหกก็ได้ — แกนบลัฟของเกม) → เข้าสู่ resolution (ข้อ 4)
- **ข้าม** → รับเงินเดือนจากระบบ **฿300** (`config.salarySkip`)
- เปิดทิ้งการ์ดครบ 2 ใบแล้ว = โจมตี/เคาน์เตอร์ไม่ได้ถาวร (ยัง challenge/ลงทุน/ชนะได้)

### เฟส 2 — เฟสแมพ (จับเวลาจริง 60 วินาที — `config.mapSeconds`)
- เดินบนแมพ + ทำธุรกรรมทุกอย่าง (ซื้อขาย/ธนาคาร/ข่าว/ส่องพอร์ต) ภายในเวลานี้
- **นาฬิกาเดินต่อแม้เปิดหน้าต่างตึก** — เวลาอ่าน/กดซื้อขายกินเวลาใน 60 วิจริง
- หมดเวลา / กดจบเทิร์น = จบเทิร์นทันที
- หน้าจอ = แมพเต็มจอ UI ลอยทับ (โปรไฟล์เทิร์นซ้ายบน · ⛶ เต็มจอ/🎯 จูนตึก/นาฬิกาขวาบน · hint+สัญญาณบอทซ้ายล่าง · ปุ่มจบเทิร์น/⚔️/ข้ามขวาล่าง)

### ท้ายรอบ (ทุกคนเล่นครบ — `js/resolve.js` เงียบ ไม่มีจอสรุป)
ลำดับ: ข่าวครบกำหนด → ราคาหุ้นขยับ×ตัวคูณข่าว → ปันผลหุ้น → ตลาดสินทรัพย์ขยับ+จ่ายค่าเช่า/ดอกเบี้ย/ปันผล → ข่าวคูณเข้าเหรียญ/ทอง → ดอกเบี้ยธนาคาร → GreenHub ปิดเดือน → ตัวนับสถานะลด → เฉลยข่าวเข้าฟีด

## 3. การ์ดอาชีพลับ 8 อาชีพ (`js/professions.js`)

- กอง **24 ใบ** (8 อาชีพ × 3) แจก**คนละ 2 ใบลับ** · อ้างอาชีพที่ไม่มีก็ได้ (โกหก)
- **วงจรเคาน์เตอร์**: สื่อ → วาฬ → ชอร์ต → เสือ → แบงค์ → มาเฟีย → ก.ล.ต. → นักสืบ → วนกลับสื่อ (อาชีพถัดไปในวงจร = ผู้เคาน์เตอร์ท่าของอาชีพก่อนหน้า — `SC.counterOf`)

| อาชีพ | ท่า | ผล (ตัวเลขจาก `config.js`) |
|---|---|---|
| 📡 เจ้าพ่อสื่อ | ยัดข่าวปลอม | ดูการ์ดข่าวสุ่ม 1 ใบของเป้า แล้วเลือกสลับกับใบบนสุดของกองหรือวางคืน — เป้าไม่รู้ว่าโดนใบไหน (เป้าไม่มีข่าว = ผู้โจมตีจั่วข่าวฟรี 1 ใบแทน) |
| 🐋 วาฬคริปโต | เทใส่ | คริปโตในพอร์ตเป้าหาย **20%** — ครึ่งหนึ่งของมูลค่าที่หาย (10%) เข้ากระเป๋าวาฬเป็นเงินสด อีกครึ่งหายเข้าระบบ (เป้าต้องถือคริปโต) |
| 📉 สายชอร์ต | ชอร์ตพอร์ต | เป้าจ่าย **10% ของหมวดสินทรัพย์ที่ถือมูลค่าสูงสุด** (นับเงินฝากเป็นหมวดด้วย) เพดาน **฿1,500** · เงินสดไม่พอ = บังคับขาย −10% |
| 🐅 เสือนอนกิน | ฮุบที่ดิน | บังคับซื้ออสังหาของเป้า 1 หน่วย (ระบบเลือกหน่วยแพงสุดที่ผู้โจมตีจ่ายไหว) ที่ราคาตลาด **−15%** · เป้าปฏิเสธได้โดยจ่ายค่ายอมความ **฿500** ให้ผู้โจมตี |
| 🏦 นายแบงค์ | บีบหนี้ | เป้าชำระหนี้คืนระบบทันที **50%** ของยอดคงค้าง (เงินสดไม่พอ = บังคับขาย −10% จนครบ) — เป้าต้องมีหนี้ |
| 🕶️ มาเฟียเงินกู้ | ข่มขู่ | เป้าเลือก: จ่ายผู้โจมตี **฿1,000** หรือ**เปิดการ์ดอาชีพ 1 ใบให้ทุกคนเห็นถาวร** (faceUp — ยังใช้ได้ แค่ไม่ลับ) · เงินไม่ถึงพัน = บังคับเปิด |
| ⚖️ ก.ล.ต. | อายัดบัญชี | เทิร์นถัดไปของเป้า**ห้ามทุกธุรกรรม+ห้ามโจมตี** เดินแมพได้อย่างเดียว (ยัง challenge/counter ได้) |
| 🕵️ นักสืบไซเบอร์ | เจาะระบบ | เลือกโหมด: ขโมยเงินสด **฿800** หรือแอบดูการ์ดอาชีพเป้า 1 ใบ + สรุปพอร์ตเต็มของเป้า (เห็นคนเดียว) |

- เงื่อนไขเป้า validate จากความจริงใน `SC.attacks.canTarget` (วาฬ→เป้ามีคริปโต · ชอร์ต→เป้ามีสินทรัพย์ · เสือ→เป้ามีอสังหา+ผู้โจมตีจ่ายไหว · แบงค์→เป้ามีหนี้ · ที่เหลือเลือกใครก็ได้)
- **บังคับขาย** (`forcedSell`): ขายที่ราคาตลาด −10% ไล่ลำดับ ฝาก(ถอนเต็มไม่โดนหัก) → หุ้น → กองทุน → พันธบัตร → ทอง → คริปโต → อสังหา

## 4. Resolution / Duel (`js/attacks.js` — `SC.duel`)

1. ผู้โจมตีประกาศท่า+อ้างอาชีพ+เป้า
2. **เป้าตอบภายใน 10 วิ** (`config.targetResponseSec`) — หมดเวลา = ยอมอัตโนมัติ:
   - **ยอม** → ท่าทำงาน
   - **Challenge** ("ไม่เชื่อว่ามีอาชีพนี้") — ผู้อ้างมีจริง = พิสูจน์ เปิดใบนั้นสับกลับกอง จั่วใบใหม่ (ตัวตนกลับมาลับ) ผู้ท้าแพ้ · ไม่มีจริง = ผู้อ้างแพ้
   - **Counter** โดยอ้างอาชีพเคาน์เตอร์ (ตามวงจร — อ้างมั่วได้เช่นกัน) → ผู้โจมตี challenge กลับได้อีก 1 ชั้น
3. **Challenge ลึกสุด 2 ชั้นจบ** ไม่มีซ้อนต่อ · บุคคลที่สามแทรกไม่ได้
4. บทลงโทษ/ค่าตอบแทน:
   - **แพ้ challenge** = จ่ายคู่กรณีโดยตรง **฿800** (ถ้ากำลังติดเครดิตพัง = **฿1,600**) + **เปิดทิ้งการ์ด 1 ใบถาวร** + ติด**เครดิตพัง**
   - **เคาน์เตอร์สำเร็จ** (ไม่โดน challenge หรือ challenge แล้วรอด) = ท่ายกเลิก + ผู้โจมตีจ่ายเป้า **฿500**
- **ไม่มีกองกลาง** — เงินทุกรายการไหลตรงระหว่างคน หรือเข้า/ออก "ระบบ"
- เงินสดไม่พอจ่าย = บังคับขาย −10% อัตโนมัติจนพอ (หรือของหมด)

### สถานะ

| สถานะ | ผล |
|---|---|
| **เครดิตพัง** (`brokenCredit`) | กู้ธนาคารไม่ได้ + ไม่ได้ดอกเบี้ยเงินฝาก + แพ้ challenge ซ้ำโดนปรับ ×2 — มีผลจริง **2 รอบ** (`config.brokenCreditRounds: 2` — โค้ดเซ็ต 2+1=3 เพราะตัวนับโดนหักท้ายรอบที่โดนทันที) |
| **อายัด** (`frozenNext`→`frozen`) | เริ่มมีผลเทิร์นถัดไป: เปิดหน้าต่างตึกไม่ได้ทุกตึก เดินได้อย่างเดียว |

## 6. ตลาดการลงทุนทั้งหมด

**สูตรราคากลางทุกตลาด** (random walk — เดินรอบละครั้งท้ายรอบ):
`ราคาใหม่ = ราคาเดิม × (1 + drift + สุ่ม(−vol..+vol))`
drift (แนวโน้มจริง) **ซ่อนจากผู้เล่น** — ต้องเดาจากคำใบ้ในตึกข้อมูล · กราฟทุกตัว seed ประวัติย้อนหลัง 12 จุดตอนเริ่มเกม · วอลุ่มหุ้นเป็นตัวเลขสุ่มตกแต่ง ไม่มีผลจริง

### กลไกหลัก
- **AP 2 แต้ม/เทิร์น** — ซื้อของกรีน/อัปเกรด/สมัครใบรับรอง ใช้ 1 AP ต่อครั้ง (รีเซ็ตทุกเดือน)
- **ยอดขาย/เดือน** = min(ฐานลูกค้า × ตัวคูณตลาด × (1 + ชื่อเสียง/500) × โทษข่าวลบ, เพดานผลิต)
- **กำไรเข้าเงินสด** = ยอดขาย × กำไร/หน่วย × ตัวคูณราคา + รายได้เสริม + ออเดอร์ B2B − fixed (ขั้นต่ำ ฿40)
- ฐานลูกค้าโตธรรมชาติ +3%/เดือน · XP = กำไร/20 → เลเวลธุรกิจ
- **ดัชนีคาร์บอน** เริ่ม 100% (ตาม footprint ธุรกิจ 4 หมวด: พลังงาน/วัสดุ/ของเสีย/ขนส่ง) — ไอเท็มกรีนลดแบบคูณซ้อนภายในหมวด
- **โอกาสดราม่า/เดือน** = min(45%, 15% × ดัชนี/100 × สายตาสังคม/100) → ม็อบ/แฮชแท็ก/รีวิวแย่ (ยอด −15%, ชื่อเสียงลด · มี Sustainability Officer = ผลกระทบครึ่งเดียว) · ฝั่งดี: ร้านสะอาด/ดังพอ มีโอกาส 25% ได้ข่าวบวก
- **แคตตาล็อกลงทุนกรีน 17 ชิ้น** ใน 5 หมวด (฿40–฿1,600 หรือสัญญารายเดือน) — LED, Inverter, โซลาร์เช่า/ซื้อ, RE100 (ล็อกพลังงานเหลือ 5 จุด), ถังแยกขยะ, ถังหมักปุ๋ย, แพ็กเกจชานอ้อย, ระบบมัดจำ, รวมรอบส่ง, จักรยานไฟฟ้า, แชร์ EV, Officer, Carbon Tracking ฯลฯ — บางชิ้นลด fixed/เพิ่มรายได้เสริม/บูสต์ลูกค้า
- **อัปเกรดธุรกิจ 5 อย่าง**: เครื่องที่ 2 (฿600, เพดาน +150) · จ้างพนักงาน (฿120/เดือน, +80) · ตกแต่งร้าน (฿400, ชื่อเสียง+10 ลูกค้า+10%) · สินค้าซิกเนเจอร์ (฿200, กำไร+฿2) · สาขา 2 (฿3,000, ลูกค้า×1.8 fixed+฿300)

## 12. สิ่งที่จงใจ "ไม่มี" ในเวอร์ชันปัจจุบัน

- ไม่มี multiplayer (บอทเท่านั้น) · ไม่มีเซิร์ฟเวอร์ — ทุกอย่างรันในเบราว์เซอร์
- ไม่มีหน้าสรุปท้ายรอบ / เฟสตัดสินใจแยก — ท้ายรอบประมวลผลเงียบ
- ไม่มีระบบ "เวลาอ่านข้อมูล" (read-bar) — แต่เวลาเปิดหน้าต่างกินเวลาเทิร์นจริง
- ไม่มีป้ายชื่อตึกบนแมพ / ไม่มี hover ยกตัวตึก / ไม่มีวงแหวนสถานะใต้ตึก (เหลือ badge+เรืองแสง)
- ตึก City Hall (trend) ถูกถอดออกจากเกมแล้ว — น้ำพุเป็นฉากเฉยๆ
- ข่าวลือไม่มีผลราคาจริง (เครื่องมือจิตวิทยาล้วนๆ)
- ปุ่ม Auction House / Property Market / Upgrade Building ใน Real Estate = ยังไม่เปิดใช้ (toast)
- อัปเกรดความปลอดภัยตู้เซฟทอง = ใช้จริงแล้วกับเหตุการณ์ `goldHeist` (ระดับ 3 = กันโจร + รับรางวัล ฿400)

## 13. ระบบเหตุการณ์ (Event System — `js/eventCatalog.js` + `js/events.js`)

ระบบเหตุการณ์สุ่มทั้งเมือง — ยิงต้นรอบก่อนเทิร์นแรก (hook `SC.events.onRoundStart` ใน `SC.flow.startRound`) · ค่าจูนรวมที่ `SC.eventsCfg` ที่เดียว · **เกมเล่นได้ครบแม้ลบโฟลเดอร์ `assets/events` ทั้งอัน** (ทุกรูปมี fallback emoji + กล่องสีตาม tier)

### จังหวะเวลา (ต้นรอบ)
1. ประมวล **telegraph** ค้าง → 70% ยิงจริง / 30% เก้อ ("ตลาดปกติดี ขออภัยที่ทำตกใจ")
2. ประมวล **chain** ครบ delay → ทอย p ยิง/ทิ้ง (เช่น oilShock→inflation→rateHike · memeSqueeze→memeDump · dogeTweet→dogeDelete · flood→floodRecovery)
3. ยังไม่มีเหตุการณ์รอบนี้ → ทอยใหม่ `baseChance` (ชิล 0.35 / ปกติ 0.55 / โกลาหล 0.85) · **pity: รอบก่อนว่าง → การันตียิงทันที** (ไม่มีช่องว่าง >2 รอบ) · crisis ×1.3
4. tier สุ่มตามน้ำหนัก light 60·mid 28·heavy 10·legend 2 (tier ว่าง → ลดขั้น) · เลือกในกลุ่มตาม `w`
5. **heavy/legend ฝั่งลบ** (`neg`) = ไม่ยิงทันที แต่ประกาศ **การ์ดเตือน 🔮** ยิงจริงรอบหน้า
6. เหตุการณ์อื่นยิงทันที: ผลกายภาพ (ตึกปิด/เงินสด/สถานะ) มีผลเดี๋ยวนั้น · ผลราคาเป็น pending คูณท้ายรอบ

### เงื่อนไข: รอบ 1 เฉพาะ light/mid · legend เฉพาะรอบ 3–9 · รอบ 10 ห้าม heavy/legend-neg · cooldown เดิม 3 รอบ · heavy 2 รอบติดห้าม · ตึกปิดพร้อมกันสูงสุด 1 หลัง · เจาะบุคคลฝั่งลบห้ามลงคนเดิม 2 ครั้งติด

### Regime (Markov ซ่อนจากผู้เล่น — เดินท้ายรอบใน resolve)
`normal 🌤️ / boom 🚀 / slump 🌧️ / crisis ⛈️` — บวกเข้า drift ทุกสินทรัพย์ + คูณ vol (boom หุ้น +2% · crisis หุ้น −6% คริปโต −12% **ทอง +4%**) · ดัชนีฟองสบู่ boom ยาว → โอกาส crisis เพิ่ม · **ข่าวบรรยากาศ**ต้นรอบใบ้ regime (แม่น 80%/หลอก 20%)

### ราคาแบบมี correlation (`SC.events.stepAsset` — ใช้ทุกตลาดยกเว้นพันธบัตร)
`p' = p × (1 + drift + regimeDrift + eventDrift + β·M·vol·wild + idio·vol·0.6) × eventMult × newsMult` · M = ตัวขับตลาดร่วม (ผลรวม rand 4 −2) · wild 6% หางอ้วน ×2 · β: หุ้น 0.6–1.6 · คริปโต 1.8–2.6 · **ทอง −0.4 (M<−1 → −0.9 safe haven)** · อสังหา 0.35 (lag 1 รอบ) · volatility clustering หลัง heavy/legend (vol ×1.5 สองรอบ) · เพดานกันหลุดโลก (>3× หรือ <0.25× ราคาเริ่ม → ดึง drift กลับ)

### แคตตาล็อก ~40 เหตุการณ์ (6 หมวด): มหภาค (rateHike/rateCut/oilShock/recession/stimulus/tourismBoom) · รายสินทรัพย์ (earnings/whaleDump/dogeTweet/cryptoCrackdown/geoTension/**memeSqueeze**/megaProject/propertyBubble) · แมพ (**meteor** ปิดตึก+คัตซีน/goldHeist/flood/blackout/pigeonBank/roadCollapse/festival) · เจาะบุคคล (dogSteal/oldPants/lottery/catKeyboard/taxAudit/taxRefund/phoneDrop/uncleTip) · GreenHub (greenInfluencer/ratLive/greenContest/supplierHike) · Meta (botToilet/ufo/newsTypo/fortuneTeller/calmTown)

### ผลกระทบ (fx types) เชื่อมระบบเดิม
- **ตึกปิด** (`closed`): เข้าไม่ได้ทั้งผู้เล่น/บอท (turn.js + windows.js + botBrain) · วาดสไปรต์ `tower_broken` หรือ brightness 0.45 + ป้าย "ปิดซ่อม N รอบ" (map.js) · **`gold` ปิดไม่ได้เด็ดขาด** · ทุกท้ายรอบที่มีตึกปิด ทองได้ drift +3%/ตึก (`goldClosedDrift`)
- **ดอกเบี้ยธนาคาร** อ่านจาก `s.events.rates` (rateHike/rateCut) · **ค่าเช่า×rentMult / ปันผล×divMult** (markets/resolve) · **GreenHub custMult/fixedMult + AP lock ตอน startup ปิด** (greenhub) · **เวลาเฟสแมพ** mapTime (blackout 30 วิ / phoneDrop 40 วิ) · **speedMult** เดินช้า (roadCollapse) · goldTheft/forceBuy/giveInsiderCard/flipLastPublicNews/botSkipMap/prophecy/choice(ratLive)
- ประกาศ: overlay หนังสือพิมพ์ปลิวเข้ามา (paper_good/bad + ไอคอน + พาดหัวไทย) → หมุดฟีด 📌 · meteor มีคัตซีนพุ่ง+วาบ+เขย่าจอ · log เข้า `s.timeline` (จอจบเกมแสดงในไทม์ไลน์)

### บอท (`botBrain.js`): เห็นเหตุการณ์เป็นข้อมูลสาธารณะ · ไม่เดินเข้าตึกปิด · นักพนันซื้อ MEME ตอน squeeze · เต่าย้ายเข้าทอง+ฝากเมื่อเห็น telegraph/crisis · หลัง legend แวะตึกข่าว

### ตัวเลือกความเข้มข้น (หน้าเริ่มเกม): ชิล / ปกติ / โกลาหล (เก็บ `SC.eventsCfg.intensity` — ปรับ baseChance + ขนาดผลราคา)

### Debug: `SC.events.debugFire('meteor')` · `SC.events.debugRegime('crisis')` · `SC.events.state()` · `SC.events.simulate(1000)` · URL `?evt=<id>` (ยิงต้นรอบ 1)
> ผล `simulate(1000)`: เฉลี่ย ~6.7 เหตุการณ์/เกม (เป้า 6–8) · ช่องว่างยาวสุด 1 รอบ (เป้า ≤2) · สัดส่วน tier เรียง light>mid>heavy>legend (mid สูงกว่าน้ำหนักเล็กน้อยจาก chain ที่เป็น mid เป็นส่วนใหญ่ · heavy ต่ำกว่าจาก neg-heavy ที่ต้อง telegraph)

### Assets: `assets/events/` (icons/fx/tower_broken/paper) — `tools/normalize_event_assets.py` ก็อปชื่อไฟล์ที่ user ส่งให้ตรง id ใน catalog (ไม่ลบต้นฉบับ)
